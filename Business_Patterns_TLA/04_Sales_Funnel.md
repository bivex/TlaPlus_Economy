# Паттерн 04: Воронка Продаж и Цифровой Бизнес

**Экономическая база:** Маркетинг, конверсия, монетизация лояльности. Эта модель была разработана для анализа экономики внимания в современных интернет-бизнесах.

## Суть для Бизнеса
Путь клиента никогда не бывает мгновенным. Нельзя купить холодный трафик и сразу превратить его в максимальный доход. Бизнес работает как система сообщающихся сосудов: 
Контент ➔ Холодный трафик ➔ Теплая аудитория ➔ Подписчики (Лиды) ➔ Покупатели.

Узкое горлышко (bottleneck) на любом из этих этапов ломает всю экономику. Если нет контента — нет трафика. Если нет прогрева — трафик не конвертируется в лиды.

## Как моделировать в TLA+

Вы создаете "ведра" (variables) для каждого этапа воронки. Действия (actions) перекачивают людей из одного ведра в другое с определенным коэффициентом конверсии.

### Пример кода:

```tla
VARIABLES content, cold_traffic, warm_audience, leads, revenue

(* 1. Производство контента (статьи, видео, реклама) *)
CreateContent ==
    /\ content' = content + 1
    /\ UNCHANGED <<cold_traffic, warm_audience, leads, revenue>>

(* 2. Привлечение трафика (алгоритмы дают показы на контент) *)
GetTraffic ==
    /\ content > 0
    /\ cold_traffic' = cold_traffic + (content * 100)
    /\ UNCHANGED <<content, warm_audience, leads, revenue>>

(* 3. Прогрев (часть холодного трафика проникается доверием) *)
WarmUp ==
    /\ cold_traffic >= 10
    /\ cold_traffic' = cold_traffic - 10
    /\ warm_audience' = warm_audience + 2 \* Конверсия 20%
    /\ UNCHANGED <<content, leads, revenue>>

(* 4. Лидогенерация (теплая аудитория оставляет контакты) *)
CaptureLeads ==
    /\ warm_audience >= 5
    /\ warm_audience' = warm_audience - 5
    /\ leads' = leads + 1 \* Конверсия 20%
    /\ UNCHANGED <<content, cold_traffic, revenue>>

(* 5. Монетизация (продажа продукта лидам) *)
SellProduct ==
    /\ leads > 0
    /\ leads' = leads - 1
    /\ revenue' = revenue + 100 \* Высокий чек
    /\ UNCHANGED <<content, cold_traffic, warm_audience>>
```

## Как проверять модель (Model Checking)
Эта модель идеально подходит для поиска "идеального баланса" при ограниченных ресурсах (связка с Паттерном 01). Вы можете добавить переменную `budget` и заставить алгоритм выбирать: потратить $10 на создание нового контента или на прямой ретаргетинг теплой аудитории. TLC найдет путь, который генерирует максимум `revenue` за минимальное число шагов.
